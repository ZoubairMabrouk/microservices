## 🎯 8 Jobs du Pipeline Détaillés

### 1️⃣ **Code Quality** (~2 min)
```
├─ dotnet format --verify-no-changes
├─ SonarCloud Scan (si token configuré)
└─ Code metrics & analysis
```

**Output**: Code quality report sur GitHub  
**Status**: ✅ Pass/⚠️ Warning/❌ Fail

---

### 2️⃣ **Unit Tests** (~3 min)
```
├─ dotnet test (21 tests)
├─ Couverture OpenCover (coverage.opencover.xml)
├─ Publication résultats GitHub
└─ Upload à Codecov
```

**Output**: Test report, coverage metrics  
**Status**: ✅ All pass/❌ Some fail

---

### 3️⃣ **Security Checks** (~2 min)
```
├─ Snyk vulnerability scan (si token)
├─ dotnet list package --vulnerable
└─ Security report
```

**Output**: Security findings  
**Status**: ✅ No vulnerabilities/⚠️ Found/❌ Blockers

---

### 4️⃣ **Docker Build** (~5 min)
```
├─ docker/setup-buildx-action
├─ Build multi-stage
├─ Push à GHCR (si pas PR)
├─ Métadonnées & tags
└─ Cache management
```

**Output**: Image publiée sur GHCR  
**Tags**: latest, branch, sha, version  
**Status**: ✅ Published/❌ Failed

---

### 5️⃣ **Versioning** (~1 min) [MAIN ONLY]
```
├─ Generate version (YYYY.MM.DD.HHMM)
└─ Create GitHub Release avec notes
```

**Output**: GitHub Release créée automatiquement  
**Tags**: vYYYY.MM.DD.HHMM  
**Status**: ✅ Released/⏭️ Skipped (non-main)

---

### 6️⃣ **Deploy Production** (~5 min) [MAIN ONLY]
```
├─ Backup containers actuels
├─ Pull latest code (git fetch/reset)
├─ Authentifier GHCR
├─ Pull Docker image
├─ Health checks
├─ Arrêter anciens containers
├─ Démarrer nouveaux containers
├─ Vérifier health
├─ Rollback automatique si erreur
└─ Cleanup dangling images
```

**Output**: Service mis à jour en production  
**Health**: http://VPS:8080/health  
**Status**: ✅ Deployed/⏮️ Rolled back/❌ Failed

---

### 7️⃣ **Deploy Staging** (~3 min) [DEVELOP ONLY]
```
├─ Pull latest code
├─ Authenticate GHCR
├─ Pull Docker image
└─ Restart containers
```

**Output**: Service mis à jour en staging  
**URL**: http://STAGING:8086  
**Status**: ✅ Deployed/❌ Failed

---

### 8️⃣ **Notifications** (<1 min) [ALWAYS]
```
├─ Slack notification (webhook)
└─ Discord notification (webhook)
```

**Output**: Notifications dans Slack/Discord  
**Status**: ✅ Sent/⏭️ Skipped (no webhooks)

## 🚀 Quick Setup (Récapitulatif)

### 1. Valider Localement (2 min)
```powershell
# Windows
.\.github\scripts\validate-pipeline.ps1

# Linux/Mac
bash ./.github/scripts/validate-pipeline.sh
```

### 2. Configurer Secrets (3 min)
```bash
gh secret set GHCR_TOKEN --body "YOUR_TOKEN"
gh secret set VPS_HOST --body "192.168.1.100"
# ... autres secrets
```

### 3. Créer Environnements (1 min)
```bash
gh environment create production
gh environment create staging
```

### 4. First Push (1 min)
```bash
git add .
git commit -m "chore: add CI/CD pipeline"
git push origin main
```

### 5. Suivre Pipeline (2 min)
